修复dpkg返回错误代码1的问题
2024-11-15 # linux

apt安装中断后出现Sub-process /usr/bin/dpkg returned an error code (1)
之后无论用apt install -f 或apt autoremove 都不行。
sudo dpkg --configure -a
sudo apt install -f
sudo apt remove --purge package_name
sudo apt autoremove

grub2修改启动项
2024-11-14 # linux

要修改GRUB2的默认启动项，可以编辑/etc/default/grub文件
GRUB_DEFAULT 修改默认项
GRUB_TIMEOUT 修改等待时间
修改GRUB2启动项的背景和主题，可以将背景图片复制到/boot/grub目录下，并替换原有的背景图片。可以通过编辑/etc/default/grub文件来设置背景图片的路径
GRUB_BACKGROUND
添加自定义的启动项，可以通过编辑/etc/grub.d/40_custom文件来实现，添加新的menuentry条目

docker更换存储位置
2024-11-11 # docker

docker info | grep "Docker Root Dir"
默认在/var/lib/docker
systemctl stop docker
mv /var/lib/docker /data/docker
ln -sf /data/docker /var/lib/docker
重启docker
systemctl start docker
