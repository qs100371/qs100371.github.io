cloudflare zero trust tunnels 和 lucky stun穿透的比较

Posted on 2024-12-28，1 min read

cloudflare zero trust tunnels 和 lucky stun都能进行内网穿透，用域名进行直接访问。
cf tunnels在使用时，浏览器地址栏域名保持不变，而stun方法地址栏会由域名跳转到ip加端口，并且浏览器上会提示不安全（非https），穿透ip和端口也无法保持。

stun穿透

除了地址栏显示让人不适之外，速度一直没有问题。有一点，在ip变化的时候，cf域名响应有时会有滞后（短时无法访问）。

cf tunnels

最近经常打不开


使用 CF Tunnel 速度上限是 32Mbps 左右，延时可能会略高于直接使用 CDN，但仍能保持在可接受的范围内。隧道速度取决于用户到 CF 数据中心 的距离和网络状况。CF 隧道无端口限制。
但是有个相当致命的缺点：Cloudflare到中国大陆的网络情况不佳，分配到的EndPoint基本都在美西，速度慢不说，还经常炸。
